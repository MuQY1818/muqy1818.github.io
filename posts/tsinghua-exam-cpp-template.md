## 为什么值得专门做一份打印板子

平时的刷题笔记都在 Obsidian 里（图论、DP、数学、数据结构分门别类），但机考现场没有电脑给你翻笔记，只有纸。开卷考的核心矛盾是：**考场时间应该花在思考上，而不是回忆 `priority_queue` 的比较器怎么写**。

所以板子的目标很明确——把"会忘但不该花时间想"的内容全部固化到纸上：

- STL 的确切用法（`unordered_map` 自定义哈希、`tuple` 的比较规则、`lower_bound` 的返回值）
- 高频模板的完整可运行代码，抄下来改改输入就能交
- 看到题目时的选型依据（数据范围、题型信号）
- 针对清华题型的考场执行策略

## 组织方式：STL 速查在前，算法模板在后

板子按「C++17/STL 速查在前、算法模板在后」组织，用 `ctexart` + `xelatex` 编译（两遍生成目录），最终是一本可以装订的小册子。

每个模板统一为固定四件套：**用途、思路、复杂度、输入输出格式 + 完整可运行程序**。坚持放完整程序而不是伪代码，是因为考场上最昂贵的是调试——结构、变量名、边界全部现成的代码，改一下输入解析就能跑。比如 Dijkstra 在板子里长这样：

```cpp
#include <bits/stdc++.h>
using namespace std;

using ll = long long;
using pli = pair<ll, int>;
const int N = 200010;
const ll INF = 1LL << 62;

int n, m, source;
ll dist[N];
vector<pair<int, int>> graph[N];

int main() {
  ios::sync_with_stdio(false);
  cin.tie(nullptr);

  cin >> n >> m >> source;
  while (m--) {
    int a, b, w;
    cin >> a >> b >> w;
    graph[a].push_back({b, w});
  }

  fill(dist, dist + n + 1, INF);
  // pair 按距离、点编号依次比较，因此 greater<pli> 得到距离小根堆。
  priority_queue<pli, vector<pli>, greater<pli>> heap;
  dist[source] = 0;
  heap.push({0, source});

  while (!heap.empty()) {
    auto [distance, u] = heap.top();
    heap.pop();
    // 同一点可能多次入堆，只处理与当前最短距离一致的最新记录。
    if (distance != dist[u]) continue;
    for (auto [v, w] : graph[u]) {
      if (dist[v] <= distance + w) continue;
      dist[v] = distance + w;
      heap.push({dist[v], v});
    }
  }

  for (int i = 1; i <= n; ++i)
    cout << (dist[i] == INF ? -1 : dist[i]) << ' ';
  cout << '\n';
  return 0;
}
```

注释只写"为什么"（为什么 `greater<pli>` 是小根堆、为什么旧条目要跳过），不写"是什么"——纸面上每一行都很贵。

## 比模板更重要的是选型

§2 是我认为全板最有价值的一节，它回答的是"看到题，用哪个"的问题。

**数据范围反推复杂度**，是拿到题第一分钟该做的事：

| 数据规模 | 可接受复杂度 | 常用算法 |
| --- | --- | --- |
| \(n \le 10\) | \(O(n!)\) | 全排列、回溯 |
| \(n \le 20\) | \(O(2^n \cdot n^2)\) | 状态压缩 DP |
| \(n \le 200\) | \(O(n^3)\) | Floyd、区间 DP |
| \(n \le 2000\) | \(O(n^2)\) | LIS、LCS、朴素 DP |
| \(n \le 10^5\) | \(O(n\log n)\) | 排序、二分、树状数组、线段树 |
| \(n \le 10^6\) | \(O(n)\) | 前缀和、双指针、单调结构、筛法 |

**题型信号表**把"看到什么 → 优先想到什么"列成三十来条对照，摘几条示例：

- 多次区间加、最后统一输出 → 差分
- 左右第一个更大/更小 → 单调栈
- 区间修改 + 区间查询 → 线段树
- 负权边或"最多经过 \(k\) 条边" → Bellman-Ford
- \(n \le 20\) 且要枚举"选哪些" → 状压 + `(s-1)&s` 子集枚举

**最短路选择**单独给了一张决策表：边权全同 BFS、0/1 边权 0-1 BFS、非负 Dijkstra、负权 Bellman-Ford、多源小图 Floyd。清华真题中图论出现频率很高，按表选型即可直接确定算法。

## 真题统计与保底两题策略

板子最后一部分不是算法，是**战术**。曙梦 OJ 收录了 36 道清华机考真题（2016–2025），按位置统计完，规律非常明显：

| 位置 | OJ 难度 | 高频考点 | 代表真题 |
| --- | --- | --- | --- |
| T1 签到 | 1–2 | 计数统计、树遍历、排列枚举、快读模拟 | 众数、毕业照、拓扑分析 |
| T2 中档 | 2–4 | 大模拟、数据结构、DP、期望 | 五子棋、水滴、售货机 |
| T3 压轴 | 5–9 | 表达式解析、矩阵树定理、状压、博弈 | 表达式求值、Phi 的游戏 |

由此得到考场执行顺序（保底两题策略）：

1. 开考 10 分钟通读全部题目，T1 通常是纯计数或短模拟，30 分钟内必须 AC；
2. T2 多为大模拟：先列实体、状态、事件顺序再动手，小样例测过就交一版保底；
3. T3 直接写暴力交小数据测试点——IOI 赛制按测试点给分，暴力常拿 20–40 分；
4. 每题限 32 次有效提交、按测试点取最高分，可以放心逐版迭代，不要攒到最后一次性交。

第 4 条很多人没意识到：**IOI 赛制下"先交暴力再优化"是严格占优策略**，没有任何代价。

## 大模拟与提交前检查

T2 的大模拟没有通用算法，板子给的是方法论 + 一个完整范例（五子棋）：先列实体、状态、事件顺序、终止条件再写代码；`struct` 装实体、小函数处理单次操作，不把所有分支堆进 `main`；方向和优先级写成数组而不是散落的魔法数字；需要"同时更新"时先复制旧状态；先按小数据写直接模拟拿部分分，再替换瓶颈结构。

最后一节是提交前检查清单，均为实践中最常见的失分点：

- 先确认 0-based 还是 1-based，检查 \(n = 0/1\) 和区间端点；
- 乘法、前缀和、方案数一律 `long long`；
- `INF + w` 前先判断可达，避免溢出；
- 多组测试清空数组、邻接表、队列、堆和全局计数器；
- 自定义排序比较器相等时必须返回 `false`；
- 换算法后先与旧版暴力**对拍**，输出一致再交新版。
